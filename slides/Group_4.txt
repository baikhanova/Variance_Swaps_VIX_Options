W2 — Mathematical derivations (model framework)
4
Heston model & variance swap
Heston SDE:  dvₜ = κ(θ − vₜ)dt + σ√vₜ dWₜ
Discounted stock is Q-martingale:  E^Q[e^(−rT) S_T] = S₀
Variance swap fair strike:  K_var = θ + (v₀ − θ)(1 − e^(−κT)) / (κT)
Derivation via Itô's lemma on ln Sₜ and replication portfolio
VIX affine formula
VIXₜ² affine in vₜ under Heston:
VIXₜ² = (1/Δ) Eₜ^Q[∫ vₜ ds] = θ + (vₜ − θ) B(κ,Δ)
B = (1 − e^(−κΔ)) / (κΔ)  (mean-reversion weight)
VIXₜ = 100√(A + B·vₜ),  A = θ(1−B)
Joint calibration problem — why Heston fails?
One-factor: vₜ drives both SPX implied vol and VIX term structure simultaneously
Fitting SPX surface requires specific (κ, σ_v); VIX curve then deviates from market
Resolution: two-factor CIR (vₜ = vₜ⁽¹⁾ + vₜ⁽²⁾) or rough Heston

W2 — CIR characteristic function derivation
5
Goal:  Derive φ(u;τ) = E^Q[exp(iu·v_T) | v₀] for terminal CIR variance.
Step 1 — Feynman–Kac:  Set F(v,t) = E[e^(iuv_T) | vₜ = v]. PDE:
∂F/∂t + κ(θ−v)∂F/∂v + ½σ²v ∂²F/∂v² = 0,  F(v,T) = e^(iuv).
Step 2 — Ansatz:  Try F = exp(A(τ) + B(τ)·v), τ = T−t.  Substituting ⇒ Riccati ODE for B(τ) and linear ODE for A(τ).
Step 3 — Riccati solution:  γ = √(κ² − 2σ²iu),  D(τ) = (κ+γ) + (γ−κ)e^(−γτ)
B(τ) = 2iu(1−e^(−γτ)) / D(τ)
A(τ) = (κθ/σ²)[(κ−γ)τ − 2 ln(D(τ)/2γ)]
Step 4 — Final CF:  φ(u;τ) = exp(A(τ) + B(τ)·v₀).  Boundary check: φ(0;τ) = 1 ✓  |φ(u;τ)| ≤ 1 for real u ✓

W3 — Monte Carlo engine  (variance_mc.py)
6
Heston path simulation:  Full-truncation Euler scheme; correlated BM for (Sₜ, vₜ); variance floored at 0 to avoid negative values.
Realised variance:  RV₀₀ⱼ = (1/T)Σ (ln Sₜᵢ/Sₜᵢ₋₁)²  (annualised); compared to analytical K_var.
VIX call by MC:  Simulate VIX_T = 100√(A+B·v_T) at option maturity; payoff = max(VIX_T − K, 0); discounted expectation.
Convergence study:  Path counts 500 → 50,000; MC error ~ 1/√N as expected; CI width plotted.
Preliminary VRP proxy:  Simulated realised variance can later be compared with implied variance once market data is added; full VRP backtesting is left for the empirical stage.

W4 — Fourier/COS pricing engine  (vix_cos.py)
7
Core idea: express the expected payoff as a Fourier-cosine series using the known characteristic function of terminal variance.
COS density recovery
f(v) ≈ (2/(b−a)) Σ' Re[φ_vₜ(kπ/(b−a)) e^(−ikπa/(b−a))] cos(kπ(v−a)/(b−a))

Density integrates to 1 within 0.5%.
Expected payoff via COS
E[g(v_T)] ≈ Σ' Re[φ_vₜ(kπ/(b−a)) e^(−ikπa/(b−a))] V_k

V_k = payoff cosine coefficient.  Vectorised: O(N·M).
Pricing functions
price_vix_call_cos
price_vix_put_cos
price_vix_futures_cos
Truncation interval:  a = 0 (CIR non-negative),  b = μ + Lσ  (L = 8 std widths), based on closed-form CIR moments.

W4 — COS Pricing algorithm: step by step
8
1
Determine interval [a,b]:  a = 0,  b = E[v_T] + 8√Var[v_T]  (CIR closed-form moments).
2
Compute characteristic function:  u_k = kπ/(b−a),  φ_k = Re[φ_vₜ(u_k) e^(−iu_k a)],  k = 0, ..., N−1.
3
Compute payoff coefficients V_k:  V_k = (2/(b−a)) ∫ g(v) cos(kπ(v−a)/(b−a)) dv,  g(v) = (100√(A+Bv) − K)⁺.
4
COS summation:  E[g(v_T)] ≈ Σ' φ_k · V_k  (prime: k=0 term × ½).  Vectorised via (N×M) matrix — no loop.
5
Discount:  Price = e^(−rT) · E[g(v_T)].

W4 — Method comparison: COS vs. Monte Carlo
14
Both methods price the same VIX call. COS uses the analytical CF; MC uses simulated paths.
COS Method
Characteristic function + Fourier series
O(N·M) vectorised — runs in milliseconds
Accuracy depends on N and the truncation interval
No MC variance — deterministic
Truncation error well-controlled by std_width
Used for: call, put, futures, density, term structure
Monte Carlo Method
Simulate 10,000+ Heston/CIR paths
O(paths × steps) — slower but intuitive
Confidence interval explicit via CLT
MC standard error decreases at rate O(N⁻½)
Provides an independent benchmark for COS pricing
Used for: realised variance, VIX call pricing, and model checks
COS and MC prices are close under the same parameter set — the two methods provide cross-validation for the pricing engine.

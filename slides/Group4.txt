W1 - Literature review & research frontier

Heston model & variance swap
▶  Heston SDE:  dv_t = κ(θ − v_t)dt + σ√v_t dW_t
▶  Discounted stock is Q-martingale: E^Q[e^{−rT} S_T] = S_0
▶  Variance swap fair strike:  K_var = θ + (v₀ − θ)(1 − e^{−κT}) / (κT)
▶  Derivation via Ito's lemma on ln S_t and replication portfolio
VIX Affine formula
▶  VIX² affine in v_t under Heston:
▶  VIX²_t = E^Q_t[∫_t^{t+Δ} v_s ds / Δ] = θ + (v_t − θ) B(κ, Δ)
▶  where  B = (1 − e^{−κΔ}) / (κΔ)  (mean-reversion weight)
▶  VIX_t = 100 · √(A + B · v_t)  with A = θ(1−B)
Joint calibration problem — why Heston fails?
▶  Heston is one-factor: v_t drives both SPX implied vol and VIX term structure simultaneously
▶  Fitting SPX surface requires specific (κ, σ_v); the implied VIX curve then deviates from market
▶  Empirical evidence: large residuals in VIX futures when Heston is SPX-calibrated
▶  Resolution: two-factor CIR (v_t = v^(1)_t + v^(2)_t) or rough Heston
W2 — Mathematical derivations (model framework)

Goal: derive  φ(u; τ) = E^Q[exp(iu · v_T) | v_0]  for terminal CIR variance
Step 1: Feynman-Kac
Set F(v,t) = E[e^{iu v_T} | v_t = v].  PDE from Feynman-Kac:∂F/∂t + κ(θ−v)∂F/∂v + ½σ²v ∂²F/∂v² = 0,   F(v,T) = e^{iuv}
Step 2: Ansatz
Try F = exp(A(τ) + B(τ)·v),  τ = T − tSubstituting → Riccati ODE for B(τ) and linear ODE for A(τ)
Step 3: Riccati solution
γ = √(κ² − 2σ²·iu),   D(τ) = (κ+γ) + (γ−κ)e^{−γτ}B(τ) = 2iu(1 − e^{−γτ}) / D(τ)A(τ) = (κθ/σ²)[(κ−γ)τ − 2 ln(D(τ)/2γ)]
Step 4: Final CF
φ(u; τ) = exp(A(τ) + B(τ)·v₀)Boundary check: φ(0; τ) = 1  ✓     |φ(u; τ)| ≤ 1 for real u  ✓

W3 — Monte Carlo engine  (variance_mc.py)

Core idea: express the expected payoff as a Fourier-cosine series using the known characteristic function of terminal variance
COS density recovery
▶  Recover f(v_T) on [a,b] from φ via:
▶  f(v) ≈ (2/(b−a)) Σ' Re[φ_k] cos(kπ(v−a)/(b−a))
▶  Density integrates to 1 within 0.5%
Expected payoff via COS
▶  E[g(v_T)] ≈ Σ_k' Re[φ(kπ/(b−a)) e^{−ikπa/(b−a)}] · V_k
▶  V_k = payoff cosine coefficient (numerical integral)
▶  Vectorised k×grid matrix: O(N·M) — no Python loop
Truncation interval
▶  Lower bound a = 0 (CIR variance is non-negative)
▶  Upper bound b = μ + L·σ  (L=8 std widths)
▶  Based on closed-form CIR moments
Pricing functions
▶  price_vix_call_cos  — European call on VIX_T
▶  price_vix_put_cos   — European put on VIX_T
▶  price_vix_futures_cos — VIX futures F = e^{−rT}E[VIX_T]
W4 — Fourier engineer: COS pricing engine  (vix_cos.py)

1  Determine interval [a, b]
a = 0,   b = E[v_T] + 8·√Var[v_T]   (CIR closed-form moments)
2  Compute characteristic function
φ_k = Re[ φ(kπ/(b−a)) · exp(−ikπa/(b−a)) ]   for k = 0 … N−1φ uses noncentral chi-square CF of CIR terminal variance v_T
3  Compute payoff coefficients V_k
V_k = (2/(b−a)) ∫_a^b g(v) cos(kπ(v−a)/(b−a)) dvg(v) = max(100√(A+Bv) − K, 0)  for call;  implemented via trapezoid rule
4  COS summation
E[g(v_T)] ≈ Σ_{k=0}^{N−1} ' φ_k · V_k    (prime = k=0 term × ½)Vectorised: all k simultaneously via (N×M) matrix — no loop
5  Discount
Price = e^{−rT} · E[g(v_T)]

Both methods price the same VIX call. COS uses the analytical CF; MC uses simulated paths.
COS method
▶  Characteristic function + Fourier series
▶  O(N·M) vectorised — runs in milliseconds
▶  N=64 gives 4-decimal accuracy
▶  No Monte Carlo variance — deterministic
▶  Truncation error well-controlled by std_width
▶  Used for: call, put, futures, density, term structure
Monte Carlo method
▶  Simulate 10 000+ Heston/CIR paths
▶  O(paths·steps) — slower but intuitive
▶  Confidence interval explicit via CLT
▶  MC standard error ≈ σ/√N ≈ 0.10 for N=10K
▶  Validates COS within MC noise band
▶  Used for: VRP distribution, P&L, model check
COS price ≈ MC price within MC standard error  (absolute diff < 0.10 VIX pts)
▶  compare_vix_call_cos_with_mc helper: computes absolute and relative price difference
▶  Both methods converge to same expectation — cross-validation confirms engine correctness
